  Audio Compression Mode:      About

  Image Compression Mode：   ( Using JavaScript decoding; H264 video using h264bsd open source library, MPEG1 video and MP2 audio using jsmpeg open source library )

    1.     MJPG   Using JPEG picture flow takes up a lot of bandwidth.
But in mobile phones and other mobile devices browser is more fluent.
 
    2.     H.264(Recommend)   With H.264 compression, the bandwidth is low, but the CPU share is high.
There is a lot of delay in browsers and other mobile device browsers.
 
    3.     MPEG1   With MPEG1 compression, the bandwidth is low, but the CPU share is high.
There is a lot of delay in browsers and other mobile device browsers.
 
    4.     VP8   With VP8 compression, the bandwidth is low, but the CPU share is high.
There is a lot of delay in browsers and other mobile device browsers.
 

 Note: Please use Chrome, Firefox, Edge, Safari, Opera and other modern browsers that support the latest features, not IE and IE kernel browsers
