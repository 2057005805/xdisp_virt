About the Software


This is a remote desktop control program based on Windows platform.
It not only provides remote control, but also supports video cameras, multi-channel audio,
push stream, and multimedia entertainment functions such as watching videos and playing games on a remote desktop.
And provide a transit server program to solve the remote desktop problem of accessing the intranet.
The control end supports Windows series (including WINXP, WIN7, WIN8, WIN10).
The control side supports native clients, and also supports web clients.
Web client-supported browsers include Chrome, FireFox, Apple Safari, Microsoft Edge, Opera and other modern browsers.
IE and IE kernel browser are not supported. Web client is open platform, almost all operating system platforms support (including mobile platforms and PC platforms).
Network transmission not only supports unencrypted plaintext transmission, but also supports SSL encrypted transmission, providing security for data transmission.

Procedures and principles:
This software includes two programs (and two configuration files), xdisp_virt.exeºÍxdisp_server.exe¡£
Xdisp_virt implements the core remote desktop functions, and native or web clients can connect directly to this program,
which can also connect to the xdisp_server transit server.
Xdisp_server implements the network transfer service, runs xdisp_server on the public network server,
and xdisp_virt on the intranet connects to xdisp_server on the public network.
A native or web client can connect to xdisp_server to get all the xdisp_virt lists, so you can choose one to establish remote control.

Software uses GDI, DXGI or Mirror driver technology to capture desktop image data from Windows,
and WASAPI technology to capture the internal sound of the computer, and then uses various coding algorithms.
The audio and image coding compression, defined as a private transport protocol, is transmitted over the network to the native or web client.
Web client uses WebSocket network transmission to get the encoded image and voice, and then through JavaScript decoding,
and then draw on the page, the voice played out through the web page.
Specific principles can be described in the Windows series on CSDN for remote desktop articles.

The open source library used is as follows:
1,X264 and openh264 are used for image H264 coding.
   Mfx_dispatch and ffmpeg are used for QSV hard coding and Nvidia graphics hardcoding of H264 Intel graphics card.
2,Ffmpeg is used for MPEG4, MPEG2, MPEG1 coding, as well as some image and audio conversion, rtsp, RTMP push stream, and AAC, FLAC, AC3, MP2 audio coding.
3,Libvpx is used for encoding and decoding of VP8 and VP9.
4,Libjpeg-turbo is used for JPEG image coding and decoding, as well as RGB and YUV format conversion.
5,Libyuv is used for fast image scaling.
6,Fdk-aac and ffmpeg are used for AAC encoding, and JBIG2 is used for two value image coding and decoding.
7,X265 is used for H265 encoding.
8,Zlib, liblzma and LZO are used for lossless compression.
9,OpenSSL is used for encrypted transmission of network SSL.
10,The unqlite object database is used to save configuration parameters.
11,Jsmpeg and h264bsd, as well as some other related JS open source libraries, are used for web client JavaScript to decode images and audio.
